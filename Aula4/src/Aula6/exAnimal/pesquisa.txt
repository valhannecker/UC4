1. a sobrescrita (override) redefine o comportamento de um método herdado em uma subclasse, mantendo a mesma assinatura, enquanto a sobrecarga (overload) permite criar múltiplos métodos com o mesmo nome, mas com parâmetros diferentes.
2. sim, o TypeScript suporta sobrecarga de métodos (overload), mas não da mesma forma que o Java. Enquanto no Java você define múltiplos métodos com o mesmo nome e assinaturas diferentes (e o compilador escolhe qual chamar), no TypeScript a sobrecarga é um recurso de tempo de compilação, que é traduzido para uma única implementação em tempo de execução.
